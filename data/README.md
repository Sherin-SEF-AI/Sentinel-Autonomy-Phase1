# SENTINEL Data Directory

This directory stores runtime data generated by the SENTINEL system.

## Structure

```
data/
├── trips/              # Trip analytics data (JSON files)
│   └── trip_*.json    # Individual trip records
├── driver_scores/      # Driver scoring history
├── speed_limits_cache.json  # Cached speed limit data for GPS
└── logs/              # Application logs (if file logging enabled)
```

## Trip Data Format

Trip files are stored as `trip_<timestamp>.json` with the following structure:

```json
{
  "trip_id": "unique_id",
  "start_time": 1234567890.0,
  "end_time": 1234567900.0,
  "distance": 5000.0,
  "duration": 600.0,
  "safety_score": 85,
  "attention_score": 90,
  "events": {
    "hard_brakes": 2,
    "rapid_accel": 1,
    "lane_departures": 0,
    "collisions_avoided": 1,
    "blind_spot_warnings": 3
  }
}
```

## GPS Speed Limit Cache

The GPS tracker maintains a local cache of speed limits to reduce API calls:

```json
{
  "37.7749,-122.4194": {
    "speed_limit": 50.0,
    "road_name": "Main St",
    "road_type": "urban",
    "confidence": 0.8,
    "source": "osm"
  }
}
```

## Data Retention

- Trip data: Retained indefinitely (user can manually delete)
- Speed limit cache: Retained indefinitely (auto-updated)
- Logs: Rotate based on log configuration

## Privacy Note

This directory may contain location data if GPS tracking is enabled. Ensure appropriate
security measures are in place if handling sensitive data.
